Boundaries边界

  • Chapters 6/7 model 'standards documents'; OPF is a process-classification domain ontology and extends within the opf: namespace per Chapter 9, claiming no implementation of the full documentary entity set and changing no core definition.第 6/7 章面向“标准文献”建模；OPF 为流程分类领域本体，按第 9 章在 opf: 命名空间扩展，不声明实现文献实体全集，不改核心定义。
  • Multilingual labels fixed at source (P0-6): all 2,436 node names and descriptions carry genuine Chinese (0 zh==en placeholders, 0 residue; English source untouched, KPI names re-translated too), so Appendix A's multilingual requirement is met at source; the export keeps the zh==en filter as a defensive guard.多语言标签已在源头修复（P0-6）：全部 2436 个节点名称与描述均为 genuine 中文（zh==en 占位符 0、无中文残留 0；英文源不变，KPI 名同步重译），附录 A 多语言要求实质满足；导出仍保留 zh==en 过滤器作为防御守卫。
  • KPI and role instances ship with the later edge layer; provenance uses W3C PROV (not specified by the standard, a compatibility addition).KPI 与岗位实例随后续边层发布；溯源采用 W3C PROV（国标未规定，兼容性补充）。
